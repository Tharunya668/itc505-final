Addendum: Lights Out Game Development

  • Board Structure: I chose a 5x5 grid for the game board. The grid layout is created using CSS Grid, providing a clean and organized structure.
  • Square Styling: Each square is styled using CSS to have a width and height of 50px, a white background, and a border. The .is-off class is applied to black squares, providing a visual indicator of their state.
    •
  • Random Initialization: To ensure a solvable initial state, I employed a randomization strategy by simulating clicks on individual squares. This approach ensures that the board starts in a solvable configuration while allowing for variability in each game session.
  • Event Handling: Event listeners are used to handle square clicks. When a square is clicked, its state toggles, and the game checks for a win condition.
  • Winning Popup: Upon winning the game, a simple window.alert displays a "You win!" message. This provides a clear indication to the user that the puzzle has been successfully solved.

    • Challenges and Solutions:

  • Randomization for Solvability: The main challenge was devising a method to randomize the board while maintaining solvability. The solution involved resetting the board and simulating random clicks until a solvable state was achieved.
  • Inversion Counting: Determining the solvability of the board required counting inversions. This involves counting the number of times a higher-numbered square precedes a lower-numbered square when considering the squares in a flattened layout. The game ensures an even number of inversions for solvability.

    • Insights Gained:

  • Simulation for Solvability: Simulating clicks allowed for a controlled randomization process. By starting with an all-white board and toggling squares randomly, I ensured that the game would consistently produce solvable configurations.
  • Event Dispatching: The use of dispatchEvent to simulate clicks was a key insight. This approach allowed for a programmatic interaction with the squares, contributing to the randomization strategy.
  • CSS Class Toggle: Applying and removing the .is-off class provides a straightforward way to visually represent the state of each square. This approach enhances the user experience by clearly indicating which squares are active or "off."
    • In conclusion, the Lights Out game development process involved thoughtful consideration of the board structure, styling, randomization for solvability, and event handling. Overcoming challenges related to solvability and gaining insights into simulation techniques contributed to the successful implementation of an engaging and interactive game.
